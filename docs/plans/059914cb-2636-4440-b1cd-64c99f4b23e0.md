## Goal
Build Looma, a web app that converts YouTube links or uploaded videos into structured, reusable knowledge — refined title, 3–5 line summary, 5–10 key insights, chapter markers, and an audio-friendly narration.

## Context
- Workspace is greenfield: only `.gitignore` and `.humanize/` exist. This is a clean slate.
- Looma is explicitly **not** a blind transcriber. The product value sits in the LLM extraction step that filters filler and reorganizes content for both reading and listening.
- Single-operator, single-machine tool. No auth, no multi-tenancy, no cloud deployment in v1.
- LLM and TTS calls are billable; transcripts are bounded (≤30k chars sent to the LLM) and prompts are tight to control cost.
- Browser automation is not in scope — YouTube is fetched via `yt-dlp`, uploads come through `multipart/form-data`.

## Assumptions
- Host machine has Python 3.11+ and `ffmpeg` (install via `sudo apt-get install -y ffmpeg python3.11 python3.11-venv` is allowed).
- User has at least one valid LLM API key (Anthropic or OpenAI). Claude is used by default because of its instruction-following strength on strict JSON.
- Edge TTS is the default TTS provider (no key required, free, supports `en-US-AriaNeural`); OpenAI TTS is opt-in via `TTS_PROVIDER=openai`.
- Videos are ≤90 minutes (`MAX_VIDEO_SECONDS=5400`); uploads ≤200 MB (`MAX_UPLOAD_MB=200`).
- App runs locally at `http://127.0.0.1:8000`; no HTTPS, no auth.
- Whisper `small` model is the default; `WHISPER_MODEL` env var allows `tiny|base|small|medium|large` for accuracy/speed trade-offs.
- Frontend ships as static files served by FastAPI — no build step, no Node toolchain.

## Implementation Notes

### Architectural decisions
- **Single FastAPI process** serves both the JSON API and the static frontend from `/`. Trivial to launch (`uvicorn backend.app.main:app --port 8000`); no separate dev server.
- **Async-friendly pipeline**: CPU-bound Whisper and TTS calls run via `asyncio.to_thread` so the event loop stays responsive and the UI can poll status.
- **Double LLM validation**: the system prompt demands strict JSON (no fences, no prose), and a Pydantic model re-validates the parsed object. On schema failure, retry once with a corrective prompt; on second failure return HTTP 500 with `code=LLM_SCHEMA_ERROR`.
- **Edge TTS by default** because it is free, fast, and natural-sounding. The narrator interface is abstracted so OpenAI TTS is a one-env-var swap.
- **No DB migrations**: SQLite schema is created with `CREATE TABLE IF NOT EXISTS` at startup.
- **Disk hygiene**: `storage/files.py.cleanup_orphan_files(max_age_hours=24)` runs on app startup; `DELETE /api/jobs/{id}` also removes MP3s and DB row.
- **Startup guards**: missing `ffmpeg` or missing LLM API key ⇒ exit with a single-line actionable error (AC-14).

### LLM prompt contract (`backend/app/prompts/extract_*.txt`)
- **System prompt** instructions:
  1. Remove filler words (um, uh, like, you know, basically, actually, kind of, sort of).
  2. Reframe sentences in clear declarative form.
  3. Detect natural topic boundaries and emit chapter timestamps aligned to provided segments.
  4. Return strictly valid JSON — no prose, no markdown fences.
- **User prompt** supplies:
  - Full transcript (truncated to 30k chars with a "truncated" note).
  - Segment list `start-end: text` for chapter alignment.
  - Detected language and total duration.

### LLM schema (`backend/app/models.py`)
```python
class Chapter(BaseModel):
    start_seconds: float
    end_seconds: float
    title: str

class KnowledgeExtract(BaseModel):
    title: str = Field(max_length=120)
    summary: str
    insights: list[str] = Field(min_length=5, max_length=10)
    chapters: list[Chapter]
    narrative: str
    filler_removed: int = Field(ge=0)
```

### Risk controls
- **Long videos:** rejected at ingest via `MAX_VIDEO_SECONDS`; keeps Whisper runtime bounded.
- **Whisper model load latency:** first request pays a 5–30 s model-load penalty; surface as "loading model…" in the progress UI.
- **LLM chapter-timestamp drift:** pass segment anchors in the user prompt and post-validate that `chapters[0].start_seconds == 0` and `chapters[-1].end_seconds == duration`; snap drift to nearest segment boundary.
- **Disk fill-up:** `cleanup_orphan_files(max_age_hours=24)` on startup; DELETE removes MP3s and DB row.
- **Missing `ffmpeg` or API key:** fail-fast at startup with a one-line actionable error.

## Out of Scope
- Multi-user authentication, accounts, or shared job history.
- Cloud deployment, Docker images, CI/CD pipelines.
- Translation of transcripts into other languages.
- Speaker diarization ("who said what").
- Subtitle export (SRT/VTT) — possible later using the same Whisper segments.
- Real-time streaming of partial results.
- Mobile-native apps.
- Caching of LLM/TTS results across runs (each job is reprocessed; could be added later via a content-hash key).
- Browser automation of any kind — Looma's ingestion flows do not require a browser, and `agent-browser` is not in scope for v1.

## Deployment Suggestions & Platform Detection
Looma v1 is a Python FastAPI app that serves a small static frontend from `/`. It is best run on a single long-lived host (laptop, VPS, Fly.io machine) rather than a serverless platform, because the Whisper model and the SQLite job database are stateful local resources.

- **Not a fit for one-click Vercel/Cloudflare Pages deploys** in v1 — the pipeline depends on `ffmpeg`, `openai-whisper` (multi-GB native dependency), and a local SQLite file. A serverless platform would have to externalize all of those, which contradicts the local-first design.
- **If the user later wants a hosted Looma:** the recommended path is to containerize the backend (`Dockerfile` with `python:3.11-slim` + `ffmpeg` + Whisper model bake-in) and run it on Fly.io, Railway, or a small VM. A Vercel/Cloudflare Pages "Deploy" tab will not produce a working Looma for v1.
- **Frontend-only deploy (informational only):** the static files under `frontend/` could be lifted into a separate Vercel/Cloudflare Pages project to be served as a marketing/landing page for the local app, but that does not host the pipeline.
- **Documented in `README.md`** under a "Deployment" section so the user is not surprised that the "Deploy" tab on the right-hand panel is not the right path for v1.
